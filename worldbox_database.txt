É bom lembrar que, como o sistema escolhe o fenótipo aleatoriamente entre as opções disponíveis, uma mesma criatura pode apresentar cores diferentes em cada mundo.

O sistema de Phenotypes em WorldBox define a aparência visual das criaturas, atribuindo cores específicas com base em sua subespécie e no bioma onde nasceram. Implementado na versão 0.50, ele substituiu o antigo sistema de "Skin Colors", oferecendo uma lógica mais interativa e dependente do ambiente para a coloração das unidades.

Lógica do Sistema

· Influência do Bioma e Subespécie: As cores de uma criatura são determinadas pela combinação de sua subespécie com o bioma de nascimento. Por exemplo, ao nascer em um pântano, cobras adquirem um fenótipo verde-azulado, enquanto capivaras recebem um tom roxo-esverdeado. Cada combinação gera um fenótipo único.
· Abordagem por Força Bruta: Descobrir todos os fenótipos exige testar exaustivamente cada criatura em cada bioma e verificar suas subespécies. Esse processo manual, sem uma lista completa oficial, é um dos principais desafios para completar a coleção.
· Fenótipos como Traits Especiais: Os fenótipos são tratados como traits de subespécie que não aparecem aleatoriamente. Eles precisam ser desbloqueados ao passar o cursor sobre novos traits (indicados por um ponto de interrogação azul) ou ao completar conquistas específicas.
· Mecânica de Desbloqueio: Para registrar um fenótipo, é necessário abrir a subespécie desejada e passar o cursor sobre o trait correspondente. Traits lendários, por exemplo, são desbloqueados ao cumprir conquistas associadas.

Informações Sobre a Coleção

· Total de Fenótipos: Um jogador conseguiu coletar 37 dos 52 fenótipos disponíveis, indicando que o total da coleção é de 52 cores diferentes.
· Fenótipos Não Descobertos: Há relatos de que 10 fenótipos permanecem não descobertos pela comunidade, sugerindo que podem estar associados a condições muito específicas ou ainda não documentadas.

Considerações Finais

· Desafio e Exploração: A natureza incompleta da documentação de fenótipos reflete a filosofia do jogo de incentivar a exploração e a experimentação ativa por parte dos jogadores.
· Dica para Exploração: Para explorar o sistema, o método mais eficaz é testar sistematicamente cada criatura em cada bioma e inspecionar suas subespécies, um processo que pode ser auxiliado por ferramentas da comunidade.

    DUPLICAR = "dup"
    ESPELHAR = "mirror"
    ALEATORIO = "rand"
    PONTUACAO = "punct"

    REMOVER = "rem"
    CAPITALIZAR = "cap"
    GENERO = "gen"

# =============================================================================
# Glyph de palavras fixas
# =============================================================================

class BuildingGlyph:
    """Escolhe aleatoriamente uma palavra de uma lista."""
    def __init__(self, idx: int, palavras: List[str]):
        self.idx = idx
        self.palavras = palavras

    def resolver(self) -> str:
        if not self.palavras:
            return ""
        return random.choice(self.palavras)

# =============================================================================
# Glyph de sílabas (NOVO)
# =============================================================================

class SyllableGlyph:
    """Concatena um número aleatório de sílabas extraídas dos pools."""
    def __init__(self, idx: int, pool_silabas: List[str], min_silabas=1, max_silabas=3):
        self.idx = idx
        self.silabas = pool_silabas
        self.min_silabas = min_silabas
        self.max_silabas = max_silabas

    def resolver(self) -> str:
        if not self.silabas:
            return ""
        n = random.randint(self.min_silabas, self.max_silabas)
        escolhidas = [random.choice(self.silabas) for _ in range(n)]
        return "".join(escolhidas)

# =============================================================================
# Special Glyph
# =============================================================================

class SpecialGlyph:
    """Aplica uma transformação ao nome em construção."""
    def __init__(self, tipo: TipoSpecialGlyph, params: Optional[Dict] = None):
        self.tipo = tipo
        self.params = params if params else {}

    def aplicar(self, buffer: str, contexto: Dict) -> str:
        if self.tipo == TipoSpecialGlyph.REPETIR:
            if buffer:
                return buffer + buffer[-1]
            return buffer
        elif self.tipo == TipoSpecialGlyph.DUPLICAR:
            return buffer + buffer
        elif self.tipo == TipoSpecialGlyph.ESPELHAR:
            return buffer[::-1]
        elif self.tipo == TipoSpecialGlyph.ALEATORIO:
            if buffer and random.random() < 0.5:
                return buffer + random.choice("aeiou")
            return buffer
        elif self.tipo == TipoSpecialGlyph.PONTUACAO:
            punct = self.params.get("char", "-")
            return buffer + punct
        elif self.tipo == TipoSpecialGlyph.SEPARADOR:
            sep = self.params.get("char", " ")
            return buffer + sep
        elif self.tipo == TipoSpecialGlyph.SUBSTITUIR:
            old = self.params.get("old", "")
            new = self.params.get("new", "")
            return buffer.replace(old, new)
        elif self.tipo == TipoSpecialGlyph.REMOVER:
            char = self.params.get("char", "")
            return buffer.replace(char, "")
        elif self.tipo == TipoSpecialGlyph.CAPITALIZAR:
            if buffer:
                return buffer[0].upper() + buffer[1:]
            return buffer
        elif self.tipo == TipoSpecialGlyph.GENERO:
            gen = contexto.get("gender", "n")
            if gen == "m":
                return buffer + self.params.get("masc", "")
            elif gen == "f":
                return buffer + self.params.get("fem", "")
            return buffer
        return buffer

# =============================================================================
# Template
# =============================================================================

class TemplateNome:
    def __init__(self, id_template: str, categoria: Categoria, raca: Raca = Raca.GENERICO, cultura: str = "default"):
        self.id = id_template
        self.categoria = categoria
        self.raca = raca
        self.cultura = cultura
        self.glyphs: List[Any] = []

    def adicionar_building(self, idx: int, palavras: List[str]):
        self.glyphs.append(BuildingGlyph(idx, palavras))

    def adicionar_syllable(self, idx: int, pool_silabas: List[str], min_s=1, max_s=3):
        self.glyphs.append(SyllableGlyph(idx, pool_silabas, min_s, max_s))

    def adicionar_special(self, tipo: TipoSpecialGlyph, params: Dict = None):
        self.glyphs.append(SpecialGlyph(tipo, params))

class EntidadeNome:
    def __init__(self):
        self.categoria: Optional[Categoria] = None
        self.raca: Optional[Raca] = None
        self.cultura: Optional[str] = None
        self.template_usado: Optional[str] = None
        self.saida_final: str = ""
        self.historico_geracao: List[str] = []
        self.data_criacao: datetime = datetime.now()

# =============================================================================
# Persistência simples
# =============================================================================

class BancoNomes:
    def __init__(self):
        self.nomes: Dict[str, EntidadeNome] = {}
        self.historico: Dict[str, List[EntidadeNome]] = {}

    def salvar(self, id_entidade: str, entidade: EntidadeNome):
        if id_entidade in self.nomes:
            self.historico.setdefault(id_entidade, []).append(self.nomes[id_entidade])
        self.nomes[id_entidade] = entidade

    def recuperar(self, id_entidade: str) -> Optional[EntidadeNome]:
        return self.nomes.get(id_entidade)

    # Extração de todas as sílabas únicas
    def _extrair_todas_silabas(self):
        def silabificar(palavra: str) -> List[str]:
            padrao = r'[^aeiou]*[aeiou]+[^aeiou]*'
            partes = re.findall(padrao, palavra.lower())
            if not partes:
                return [palavra.lower()]
            return partes

        for (cat, raca), pool in self.pools.items():
            todas = []
            for lista in pool.values():
                for palavra in lista:
                    todas.extend(silabificar(palavra))
            self._silabas_por_raca[(cat, raca)] = list(dict.fromkeys(todas))  # únicas

    def get_pool(self, categoria: Categoria, raca: Raca) -> Dict[int, List[str]]:
        return self.pools.get((categoria, raca), self.pools.get((categoria, Raca.GENERICO), {}))

    def get_silabas(self, categoria: Categoria, raca: Raca) -> List[str]:
        return self._silabas_por_raca.get((categoria, raca), self._silabas_por_raca.get((categoria, Raca.GENERICO), ["a","e","i","o","u"]))

class GeradorNomes:
    def __init__(self):
        self.pools = PoolsNomes()
        self.templates = self._criar_templates_hibridos()
        self.banco = BancoNomes()

    def _criar_templates_hibridos(self) -> Dict[Tuple[Categoria, Raca], List[TemplateNome]]:
        templates = {}

        # Unit: Syllable (1-2 sílabas) + capitalizar
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC, Raca.ANIMAL, Raca.CRIATURA_ESPECIAL]:
            t = TemplateNome(f"unit_{raca.value}_syl", Categoria.UNIT, raca)
            t.adicionar_syllable(1, [], min_s=1, max_s=2)  # injetado na geração
            t.adicionar_special(TipoSpecialGlyph.CAPITALIZAR)
            # Opção de gênero: adiciona sufixo se contexto tiver gender
            t.adicionar_special(TipoSpecialGlyph.GENERO, {"masc":"", "fem":"a"})  # simplificado
            templates[(Categoria.UNIT, raca)] = [t]

        # Family: palavra fixa (já grande)
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC, Raca.ANIMAL, Raca.CRIATURA_ESPECIAL]:
            t = TemplateNome(f"family_{raca.value}", Categoria.FAMILY, raca)
            t.adicionar_building(1, [])
            templates[(Categoria.FAMILY, raca)] = [t]

        # Clan: híbrido (syllable + syllable + building)
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            t = TemplateNome(f"clan_{raca.value}_mix", Categoria.CLAN, raca)
            t.adicionar_syllable(1, [], min_s=1, max_s=1)  # prefixo silábico
            t.adicionar_syllable(2, [], min_s=1, max_s=2)
            t.adicionar_building(3, [])  # sufixo fixo
            templates[(Categoria.CLAN, raca)] = [t]

        # Village: híbrido geo fixo + cultura silábica + sufixo fixo
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            t = TemplateNome(f"village_{raca.value}_hyb", Categoria.VILLAGE, raca)
            t.adicionar_building(1, [])  # geográfico fixo
            t.adicionar_syllable(2, [], min_s=1, max_s=2)
            t.adicionar_special(TipoSpecialGlyph.SEPARADOR, {"char":" "})
            t.adicionar_building(3, [])  # sufixo
            templates[(Categoria.VILLAGE, raca)] = [t]

        # Kingdom: adjetivo + termo (fixos)
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            t = TemplateNome(f"kingdom_{raca.value}", Categoria.KINGDOM, raca)
            t.adicionar_building(1, [])
            t.adicionar_special(TipoSpecialGlyph.SEPARADOR, {"char":" "})
            t.adicionar_building(3, [])
            templates[(Categoria.KINGDOM, raca)] = [t]

        # Culture: raiz silábica + terminação fixa
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            t = TemplateNome(f"culture_{raca.value}_syl", Categoria.CULTURE, raca)
            t.adicionar_syllable(1, [], min_s=1, max_s=1)
            t.adicionar_building(2, [])
            templates[(Categoria.CULTURE, raca)] = [t]

        # Language, Religion: fixo
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            for cat in [Categoria.LANGUAGE, Categoria.RELIGION]:
                t = TemplateNome(f"{cat.value}_{raca.value}", cat, raca)
                t.adicionar_building(1, [])
                templates[(cat, raca)] = [t]

        # Alliance: fixo
        for raca in [Raca.HUMANO, Raca.ELFO, Raca.ANAO, Raca.ORC]:
            t = TemplateNome(f"alliance_{raca.value}", Categoria.ALLIANCE, raca)
            t.adicionar_building(1, [])
            t.adicionar_special(TipoSpecialGlyph.SEPARADOR, {"char":" "})
            t.adicionar_building(2, [])
            templates[(Categoria.ALLIANCE, raca)] = [t]

        # War, Item, World, Player
        templates[(Categoria.WAR, Raca.GENERICO)] = [self._simples(Categoria.WAR, 2)]
        templates[(Categoria.ITEM, Raca.GENERICO)] = [self._simples(Categoria.ITEM, 3)]
        templates[(Categoria.WORLD, Raca.GENERICO)] = [self._simples(Categoria.WORLD, 2)]
        templates[(Categoria.PLAYER, Raca.GENERICO)] = [self._simples(Categoria.PLAYER, 1)]

        return templates

    def _simples(self, cat, n):
        t = TemplateNome(f"{cat.value}_gen", cat, Raca.GENERICO)
        for i in range(1, n+1):
            t.adicionar_building(i, [])
            if i < n:
                t.adicionar_special(TipoSpecialGlyph.SEPARADOR, {"char":" "})
        return t

    def _processar(self, glyph, buffer, contexto, pool_palavras, silabas):
        if isinstance(glyph, BuildingGlyph):
            palavras = pool_palavras.get(glyph.idx, glyph.palavras)
            escolhida = random.choice(palavras) if palavras else ""
            if buffer and escolhida:
                buffer += " "
            buffer += escolhida
        elif isinstance(glyph, SyllableGlyph):
            if not glyph.silabas:
                glyph.silabas = silabas
            syl = glyph.resolver()
            if buffer and syl:
                buffer += ""  # sem espaço, junta
            buffer += syl
        elif isinstance(glyph, SpecialGlyph):
            buffer = glyph.aplicar(buffer, contexto)
        return buffer

    def gerar_nome(self, categoria: Categoria, raca: Raca, cultura: str = "default", contexto: Dict = None) -> EntidadeNome:
        contexto = contexto or {}
        template = self.templates.get((categoria, raca), self.templates.get((categoria, Raca.GENERICO), [TemplateNome("fallback", categoria)]))[0]
        pool_palavras = self.pools.get_pool(categoria, raca)
        silabas = self.pools.get_silabas(categoria, raca)

        ent = EntidadeNome()
        ent.categoria = categoria
        ent.raca = raca
        ent.cultura = cultura
        ent.template_usado = template.id
        buffer = ""
        for glyph in template.glyphs:
            buffer = self._processar(glyph, buffer, contexto, pool_palavras, silabas)
        buffer = buffer.strip()
        if buffer:
            buffer = buffer[0].upper() + buffer[1:]
        while "  " in buffer:
            buffer = buffer.replace("  ", " ")
        buffer = ' '.join([w.capitalize() for w in buffer.split()])
        ent.saida_final = buffer
        return ent

    def gerar_unidade(self, raca, unica=False, genero='n'):
        if unica:
            unicos = {
                Raca.HUMANO: ["Arthur","Merlin","Guinevere"],
                Raca.ELFO: ["Elrond","Galadriel","Celeborn"],
                Raca.ANAO: ["Thorin","Gimli","Durin"],
                Raca.ORC: ["Azog","Bolg","Gorbag"],
                Raca.CRIATURA_ESPECIAL: ["Deathwing","Smaug","Alduin"]
            }
            lista = unicos.get(raca, [])
            if lista:
                e = EntidadeNome()
                e.categoria = Categoria.UNIT
                e.raca = raca
                e.saida_final = random.choice(lista)
                e.template_usado = "unique"
                return e
        return self.gerar_nome(Categoria.UNIT, raca, contexto={"gender": genero})

    def obter_coletivo(self, tipo):
        mapa = {
            "lobo":"pack","cão":"pack","pássaro":"flock","galinha":"flock","peixe":"shoal",
            "abelha":"swarm","formiga":"colony","urso":"den","leão":"pack","elefante":"herd",
            "cavalo":"herd","humano":"army","elfo":"army","anão":"army","orc":"army","dragão":"bale"
        }
        return mapa.get(tipo.lower(), "group")

WorldBox – Simulador de Mundos em Camadas

Este documento descreve a lógica completa de WorldBox – God Simulator na sua versão atual (0.51.2). O jogo é um sandbox de simulação divina em que o jogador cria terrenos, popula o mundo com criaturas e observa civilizações emergirem, guerrearem e prosperarem, podendo intervir a qualquer momento. A simulação é organizada em camadas interdependentes que, juntas, geram um ecossistema vivo e autónomo.

Camada 1 – Base do Mundo: Terreno, Biomas e Recursos

Mapa e Terreno

O mundo é uma grelha de tiles. Cada tile pode conter água (oceano ou águas rasas), terra (areia, solo, rocha, neve, etc.) ou árvores e arbustos. O jogador constrói e modifica o terreno com os poderes do separador "Create and Shape Your World" e com as sementes e desastres do separador "Nature and Disasters".

Biomas

Um bioma é um conjunto de tiles com regras próprias. Cada bioma controla que árvores, plantas e animais aparecem, que bónus ou efeitos as criaturas recebem e até a paleta de cores das unidades. Existem 23 biomas normais e 29 se contarmos os especiais (Sand, Hill e os biomas‑spawner: Tumor, Biomass, Pumpkin e Cyber). Os biomas espalham‑se automaticamente: a cada 0,01‑0,02 meses o jogo atualiza a vizinhança de cada tile e converte os tiles adjacentes de acordo com a força e as regras do bioma. A velocidade de expansão também depende da Idade do mundo atual.

Biomas de força 6 (expandem‑se mais depressa): Arcane Desert, Candy, Corrupted, Crystal, Enchanted, Infernal, Jungle, Lemon, Mushroom, Permafrost, Savanna, Swamp, Wasteland.

Biomas de força 5: Birch, Celestial, Clover, Flower, Garlic, Grass, Maple, Paradox, Rocklands, Singularity.

· Especiais: Corrupted cresce ainda mais rápido durante a Age of Dark and Despair, pode amaldiçoar unidades, transforma mortos em esqueletos e gera fantasmas; Biomass e Cybercore comportam‑se como creep biomes – entidades estacionárias que expandem os seus tiles e geram unidades periodicamente.

Recursos Naturais

O terreno gera recursos que as civilizações recolhem e transformam.

· Madeira – Obtida de árvores.
· Pedra – Obtida de montanhas e colinas.
· Comida – Arbustos de frutos, animais caçados, peixes e culturas agrícolas.
· Metais comuns – Ferro, cobre, bronze (fundido a partir de minério).
· Metais preciosos – Prata, Mythril, Adamantine.
· Ouro – Único recurso monetário; obtido em minas e através de barcos de comércio.
· Gemas – Usadas em joalharia.
· Couro e Osso – Obtidos de animais abatidos.

Todos os recursos são armazenados nos armazéns da vila (stockpiles) e usados para construir edifícios, fabricar equipamento ou alimentar a população.

Camada 2 – Unidades: Tipos, Atributos e Genética

O que é uma Unidade

"Unit" é o termo genérico para qualquer entidade que tenha um menu próprio: criaturas, pessoas, barcos, algumas estruturas móveis. Cada unidade possui um conjunto de atributos (stats) que regem o seu comportamento.

Atributos de uma Unidade (Stats)

Atributo Descrição
Saúde (Health) Vida máxima. Quando chega a zero, a unidade morre.
Dano (Damage) Dano base por ataque.
Velocidade (Speed) Rapidez de movimento no mapa.
Esquiva (Dodge) Probabilidade de evitar um ataque.
Precisão (Accuracy) Probabilidade de acertar um ataque.
Armadura (Armor) Redução percentual do dano recebido.
Diplomacia (Diplomacy) Influencia a opinião entre reinos.
Felicidade (Happiness) Varia de -100 a 100; afeta produtividade e risco de rebelião.
Fome (Hunger) Desce 1 ponto a cada 1,6 meses. Ao atingir 0, a unidade perde vida por inanição.

Os atributos são afetados por traits, idade, cultura, equipamento e estado de saúde.

Traits (Características)

Traits são modificadores temporários ou permanentes que alteram os atributos e o comportamento. Existem mais de 200 traits, divididos em sete categorias principais: Body, Mind, Spirit, Acquired, Fun, Miscellaneous e Special.

· Hereditariedade: traits indicam se podem ser herdados; se não houver percentagem explícita, a probabilidade de herança é calculada como Chance de Nascimento × 10.
· Atribuição: traits são dados ao nascer (pela genética ou aleatoriamente), ao interagir com o mundo (comer certos alimentos, sofrer uma maldição) ou pelo jogador através do editor de traits.
· Efeitos especiais: alguns traits concedem habilidades ativas (voar, nadar, curar‑se) ou impõem restrições (não poder reproduzir‑se, não poder falar).

Genética e Reprodução

Cada espécie tem um genome base. As subespécies variam desse genome e os seus valores são comparados com o base, surgindo a verde (acima), amarelo (igual) ou vermelho (abaixo). O jogador pode editar genes com o Gene Editor.

A reprodução pode ser:

· Sexuada – Requer dois parceiros.
· Assexuada – Uma unidade gera descendentes sozinha.
· Por ataque – Algumas criaturas reproduzem‑se ao atacar (ex.: zombies, Grey Goo).

Parâmetros reprodutivos:

· Birth Rate – Número de tentativas de gerar filhos.
· Oviparidade / Viviparidade – Determinam se a unidade põe ovos ou dá à luz diretamente.
· Tempo de gestação – Período até ao nascimento.

Fenótipos

A aparência visual (cor da pele, cabelo, roupas) é determinada por fenótipos, que dependem do bioma onde a unidade nasceu e de traits genéticos.

Camada 3 – Comportamento: Tasks, Necessidades e Efeitos de Estado

Tasks (Tarefas)

Cada unidade executa tarefas (tasks) de forma contínua. As tarefas são selecionadas com base na profissão (job) e nos traits. Existem 117 tasks de comportamento registadas na wiki, agrupadas em categorias:

· Init (inicialização)
· Movement (movimento)
· Combat (combate)
· Reproduction (reprodução)
· Children (crianças)
· Socializing (socialização)
· Deliver Resources (entrega de recursos)
· Reflection (momentos de introspeção / lazer)

Exemplo – Sistema de Lazer (Reflection):

Se a subespécie tem o trait Super Positivity, executa tarefas alegres (rir, cantar, brincar).

Senão, o jogo avalia o estado da unidade e monta um Task Pool (conjunto de tarefas possíveis) com pesos:
· Se feliz (>50), adiciona Happy Laughing, Singing.
· Se infeliz (< -70), adiciona Swearing, Punch a Tree, Crying.
· Se tem o trait pyromaniac, adiciona Start Fire.
· Se é criança, adiciona brincadeiras infantis.

O jogo sorteia uma tarefa do Task Pool.

Necessidades Básicas

· Fome: diminui 1 ponto a cada 1,6 meses. A unidade procura comida quando está baixa.
· Saúde: diminui ao receber dano, sobe ao comer ou receber curas.
· Felicidade: afetada por ter emprego, casa, comida, ausência de guerra, etc.

Efeitos de Estado (Status Effects)

Efeitos temporários que modificam as capacidades da unidade. Diferem dos traits por terem duração limitada. Exemplos: Silence (impede magias), Shield (absorve dano), Curse (amaldiçoado), Madness (age aleatoriamente), Burning (perde vida por fogo), Frozen (imobilizado).

Doenças

Funcionam como efeitos de estado especiais que podem propagar‑se:

· MUSH Spores – Só pode ser introduzida pelo jogador. Quando uma unidade infetada morre, há 70% de hipótese de espalhar esporos num raio de 3 tiles.
· Zombie Infection – Transforma a vítima num zombie.
· Plague – Propaga‑se entre unidades próximas.

Doenças podem ser curadas com Divine Light, poções de cura ou habilidades como Accelerated Healing.

Camada 4 – Combate e Magia

Início do Combate

Quando duas unidades de reinos hostis (ou uma unidade hostil e uma pacífica) se aproximam, o jogo avalia se há um inimigo para atacar. A unidade escolhe entre ataque corpo a corpo, à distância ou magia, consoante o que possui e os estados ativos (Silence impede magia, por exemplo).

Tipos de Ataque

· Melee – Golpe corpo a corpo; consome stamina.
· Ranged – Dispara um projétil (flecha, pedra, etc.).
· Spell – Magia ofensiva ou utilitária.

Magias (Spells)

Algumas unidades (mages) atacam primariamente com magias. Ao atacar, tentam primeiro executar uma magia. Se a magia falhar (pela percentagem de sucesso), usam o ataque normal.

Magia Unidade Alvo Sucesso Distância Mínima Efeito
Lightning Evil Mage Inimigos 30% 6 Invoca um relâmpago médio no local.
Tornado Evil Mage Inimigos 2% 9 Invoca um tornado.
Fire Evil Mage Inimigos 20% 3 Cria fogo no local e em 3 tiles adjacentes.
Curse Necromancer Inimigos 20% 4 Amaldiçoa a área.
Random Teleport Evil/White Mage Próprio 70% (quando saúde < 60%) 6 Teletransporta‑se para terra firme aleatória.
Blood Rain Todos os mages Próprio 50% (quando saúde < 90%) 0 Cura a unidade com chuva de sangue.
Spawn Skeleton Necromancer Próprio 60% (se ≤6 esqueletos na chunk) 0 Invoca 1 esqueleto.
Shield White Mage Próprio 20% 0 Concede 30 segundos de escudo.
Cure Plague Doctor Próprio 20% 0 Lança gotas de cura sobre si mesmo.
Spawn Tree Fertilizer Druid Região 100% 0 Fertiliza árvores em redor.

Projéteis

Ao usar armas de longo alcance ou certas magias, o jogo gera projéteis. Cada um tem um efeito próprio:

Projétil Efeito
Arrow Dano perfurante.
Snowball Congela a unidade.
Rock Dano contundente.
Firebomb Incendeia a área.
Fireball Pequena explosão de fogo.
Madness Ball Causa loucura.
Plasma Ball Dano elétrico.
Shotgun Bullet Vários fragmentos.
Red/Blue/Green Orb Efeitos variados (fogo, gelo, lentidão).
Bone Dano físico; usado por necromancers.
Skull Projétil amaldiçoado.
Cannonball Destrói terreno e construções.
Acid Ball Dissolve unidades e tiles.

Morte e Pilhagem

Quando uma unidade morre, deixa cair o seu equipamento e recursos (loot). O corpo pode transformar‑se num esqueleto (se estiver num bioma Corrupted) ou num fantasma.

Camada 5 – Civilização: Vilas, Reinos e Política

Fundação de uma Vila

Quando uma criatura sapiente (com o trait Prefrontal Cortex) encontra um local adequado, funda uma vila. A primeira estrutura é a Bonfire, que cria uma nova cultura.

Construção de Edifícios

As vilas constroem automaticamente ao longo do tempo. A lógica de construção obedece a regras rígidas:

· A vila só pode planear um novo edifício a cada 1 mês.
· Não pode planear se houver inimigos na área, incêndio, ou se estiver a ser conquistada.
· Só constrói ou melhora um edifício de cada vez.
· A escolha do que construir depende da tecnologia disponível, população, limites de construção e pré‑requisitos de outros edifícios.

Lista de Edifícios Civilizacionais:

Edifício Função
Bonfire Funda a vila e a cultura; +3 de capacidade de casas.
House (Tent → Tier 1‑6) Abriga cidadãos. Cada tier suporta mais habitantes.
Town Hall Residência do rei/líder; guarda até 3 livros.
Well Decoração; +3 de capacidade de casas; pode ser usado para apagar fogos.
Windmill Cria terrenos agrícolas (fields) à sua volta.
Mine Permite que mineiros extraiam minerais; os minerais aparecem nos tiles adjacentes.
Temple +2 de capacidade de casas; local de oração.
Docks (Tier 1‑2) Constrói e estaciona barcos; +1 de capacidade de casas; serve de ponto de comércio.
Watchtower Dispara 3 flechas de cada vez contra inimigos.
Barracks Treina e mantém o exército.
Library Guarda até 5 livros.
Market Ponto de troca de recursos.

Economia e Comércio

· Ouro: único recurso monetário. Usado para construir edifícios, recrutar exércitos e manter a vila. Obtido por minas e por barcos de comércio que geram 5 de ouro ao chegar a um dock de um reino não inimigo.
· Troca entre vilas: só acontece entre vilas do mesmo reino e apenas depois de desbloqueada a tecnologia Trading. A troca é efetuada por um temporizador (supply timer) com cooldown base de 6 meses, podendo ser modificado pelas características do líder.

Reinos e Diplomacia

· Reino: conjunto de uma ou mais vilas sob o comando de um rei. Cada reino tem nome, bandeira, lema e atributos.
· Kingdom Opinions: sistema de pontuação que determina a atitude de um reino em relação a outro. Fatores: diplomacia do rei, humor do rei, distância das fronteiras, guerras em curso, inimigos comuns. A pontuação decide se os reinos vão formar alianças ou declarar guerra.
· Alianças: grupos de 2 ou mais reinos para defesa mútua. Podem surgir por diplomacia natural ou por intervenção do jogador (power Unity).
· Guerra: confronto armado entre reinos. As vilas produzem exércitos e lutam até que um lado seja destruído ou se faça a paz.

Líderes e Plots

· Kings e Líderes de Vila: governam e tomam decisões. Só membros de clãs podem executar plots importantes.
· Plots: planos de ação (declarar guerra, rebelar‑se, escrever um livro, etc.) que aparecem como ícones no mapa. O jogador pode ver o progresso, cancelar ou forçar o plot.

Clãs e Famílias

· Famílias: relações de parentesco baseadas em pais e filhos. O separador de Genealogia mostra a árvore completa.
· Clãs: linhagens reais/políticas que detêm o poder. Um clã pode ter traits de clã que afetam a lealdade e a influência.

Culturas e Línguas

· Cultura: nasce na Bonfire e propaga‑se por zonas da mesma raça. Define o nome da cultura, bandeira, traits, livros e aparência.
· Língua: associada aos livros escritos pela civilização. Línguas com o trait Melodic influenciam as tarefas de lazer (aumentam a probabilidade de cantar).

Camada 6 – Ferramentas do Jogador: Poderes e Leis do Mundo

Os 374 Poderes do Jogador

Os poderes estão organizados em 7 abas principais:

Main Tab – Visível quando o jogo abre ou quando se desselecionam as outras abas. Nele encontram‑se os menus de configuração, conquistas, conhecimento (Knowledge) e as leis do mundo (World Laws). Exibe também o botão World History para ver a linha do tempo dos eventos.

Create and Shape Your World – Ferramentas de edição do terreno e da paisagem. Exemplos: Deep Ocean, Close Ocean, Shallow Waters, Sand, Plain Soil, Forest Soil, Hills, Mountains, Shovel (up/down), Finger, Vortex, Sponge, Sickle, Bucket, Pickaxe, Spade, Axe, Demolish, Life Eraser.

Civilizations, Kingdoms and Villages – Poderes de inspeção e manipulação política. Exemplos: Village Info, Relations, Spite (faz todos odiarem um reino), Friendship (força a paz), Inspiration (cria um novo reino a partir de uma vila), Whisper of War, Discord, Wars List, War Lines, Plans and Plots, Cultures List, Culture Layer, Alliances.

Animals, Creatures and Monsters – Spawn de todas as criaturas do jogo. Abrange desde animais passivos (coelho, vaca) até monstros (dragão, zombie), passando por mages e aliens.

Nature and Disasters – Elementos naturais e desastres. Exemplos: sementes de bioma (Crystal Seeds, Lemon Seeds, Permafrost Seeds…), fertilizantes (Plants e Trees), recursos minerais (Stone, Ore Deposit, Silver, Mythril, Adamantine, Gold), fontes termais (Geyser, Acid Geyser), vulcões, nuvens (Rain, Acid, Lava, Ash, Magic, Snow, Rage).

Destruction Powers – Armas e criaturas destrutivas. Exemplos: TNT, Delayed TNT, Water Bomb, Landmine, Fuse, Fireworks, Grenade, Bomb, Napalm Bomb, Atomic Bomb, Antimatter Bomb, Tsar Bomba, Bowling Ball, Heat-Ray, Meteorite, Infinity Coin (mata metade das criaturas do mundo), Robot Santa, Grey Goo, Crab Bomb, Crabzilla.

Other various powers – Ferramentas diversas, incluindo o botão de interrogação que revela o ícone do WorldBox original e outras funcionalidades.

Camada 7 – Barcos, Equipamento e Itens Especiais

Barcos

Unidades especiais que transportam tropas, pescam ou fazem comércio. Existem três tipos principais:

Tipo Função
Transport Move exércitos e colonos.
Fishing Produz comida.
Trading Navega até docks de reinos não inimigos e gera 5 de ouro ao chegar.

Os barcos são construídos nos Docks e podem ser destruídos como qualquer unidade.

Equipamento

Armas e armaduras são fabricadas pelas civilizações com base nos recursos disponíveis. Classificam‑se em:

· Armas corpo a corpo: Sword, Axe, Hammer, Spear.
· Armas de longo alcance: Bow.
· Armaduras: Helmet, Armor, Boots.
· Acessórios: Ring, Amulet.

Os materiais seguem uma hierarquia: Wood, Stone, Copper, Bronze, Silver, Iron, Steel, Mythril, Adamantine. Couro e osso são usados para armaduras e acessórios.

Raridade: determinada pelo modifier level do item (Normal → Rare → Epic → Legendary). Itens lendários recebem um nome único.

Armas especiais: Flame Sword (dropped por Demons), Evil Staff, White Staff, Druid Staff, Necromancer Staff, Plague Doctor Staff, Alien Blaster, Shotgun.

Camada 8 – Linha do Tempo: Idades e História do Mundo

Idades (Ages)

O mundo atravessa 10 idades diferentes. Cada idade dura no mínimo 30 anos e afeta o comportamento dos biomas, a taxa de crescimento das plantas, o aparecimento de desastres e a agressividade das criaturas. O jogador pode controlar a sequência através do Age Clock.

História do Mundo (World History)

Regista automaticamente todos os eventos importantes: fundações de reinos, guerras, tratados de paz, desastres naturais, mortes de reis. A história pode ser filtrada por reis, unidades favoritas, cidades, reinos, guerras, clãs e desastres. O jogador também pode adicionar as suas próprias notas através do World History Marker.

Camada 9 – Interface do Jogador e Meta‑Sistemas

Menu Principal

Ao abrir o jogo ou sair de qualquer aba de poderes, o jogador vê o menu principal, que dá acesso a:

· World Laws – As leis que regem o mundo.
· Ages – Controlo das idades.
· World History – Linha do tempo.
· Knowledge – Registo dos 774 itens desbloqueáveis.
· Achievements – 96 conquistas (Desktop) / 86 (Mobile).
· Debug Menu – Menu de depuração (ver abaixo).
· Settings – Configurações de som, gráficos e idioma.

Menu de Depuração (Debug Menu)

Acede‑se clicando 11 vezes no ícone do hambúrguer nas configurações. Desbloqueia a conquista "God Mode". O menu contém dezenas de opções, entre as quais:

· FastSpawn – Spawn rápido de criaturas.
· Graphy – Mostra FPS, RAM e outras métricas.
· SonicSpeed – Acelera o jogo para além do 5x normal.
· ShowAmountNearArmy – Exibe o número de unidades num exército.
· ShowFoodCityText – Mostra a quantidade de comida nas vilas.
· RenderFavoriteFoods – Mostra a comida favorita de cada unidade.
· Overlay Debug Text – Sobreposição de informações de debug no mapa.
· Opções para ativar/desativar sistemas completos (Grey Goo, Conway, Greg, etc.).

Knowledge (Conhecimento)

À medida que o jogador explora o mundo, desbloqueia entradas no menu Knowledge. O menu organiza os 774 itens desbloqueáveis por categorias: Units, Items, Genes, Creature Traits, Subspecies Traits, Culture, Language, Clans. Passar o rato sobre um item novo (círculo azul) desbloqueia a entrada correspondente.

Hotkeys

Atalhos de teclado para ações comuns (copiar, colar, aumentar/diminuir velocidade, etc.).

Modding

O jogo suporta mods através do Steam Workshop, permitindo adicionar novas criaturas, biomas, poderes e mecânicas.

🧩 Complemento dos sistemas em falta

Neurons (Neurónios)

Os Neurons são a camada de inteligência artificial que rege as decisões das criaturas. Cada unidade civilizada possui uma rede de neurónios que avalia constantemente as suas necessidades e o ambiente para escolher a próxima tarefa. Funcionam como uma hierarquia de prioridades:

· Necessidades fisiológicas: fome > saúde > segurança.
· Necessidades sociais: reprodução > socialização > lazer.
· Necessidades laborais: se tem profissão, executa tarefas relacionadas (coleta, construção, combate).

Os neurónios também processam a personalidade (traits mentais) e estados emocionais (felicidade). Por exemplo, uma unidade com fome e trait Aggressive pode optar por roubar comida em vez de colher. Esta arquitetura permite comportamentos imprevisíveis mas lógicos, e é a base para a sensação de "vida" no simulador.

Menu Art

A Menu Art refere-se às ilustrações estáticas e animações de fundo que aparecem nos menus do jogo. Embora seja um elemento visual, a sua lógica reside na forma como o jogo seleciona essas artes com base no estado do mundo:

· Cada bioma, idade ou evento significativo pode desbloquear novas artes de menu.
· O jogo guarda localmente quais as artes já exibidas e alterna entre elas aleatoriamente, dando prioridade às mais recentes.
· As texturas são carregadas a partir de ficheiros .png e organizadas por categorias (Main, World, Units, etc.).

Monolith and Evolution (Monólito e Evolução)

O Monolith é uma estrutura especial que, quando colocada no mundo, acelera drasticamente a evolução das criaturas. Sua lógica de funcionamento:

· Criaturas que entram em contacto com o Monólito recebem uma grande quantidade de "pontos de evolução".
· Espécies animais próximas podem evoluir para as suas formas Evolved (ex.: um lobo torna-se Howleer) muito mais rapidamente.
· O Monólito também pode influenciar a evolução cultural, desbloqueando novas tecnologias em civilizações próximas.
· Após várias utilizações, o Monólito perde poder e pode ser recarregado com recursos raros ou intervenção divina.

Religions (Religiões)

As Religiões são um subsistema social que surge espontaneamente nas vilas quando certas condições são cumpridas:

· Um templo (Temple) precisa de estar construído.
· A vila deve ter pelo menos um membro de clã com trait Spiritual.
· A religião recebe um nome, um símbolo e um conjunto de traits que afetam os seguidores (ex.: Pacifist, Zealous).
· A religião espalha-se entre unidades da mesma cultura e pode atravessar fronteiras de reinos.
· Líderes religiosos podem aparecer e influenciar plots (ex.: declarar guerra santa, exigir tributos).
· Se a lei mundial Magic Rites estiver ativada, rituais religiosos podem desencadear eventos sobrenaturais (chuvas de sangue, pragas, milagres).

Metamorphosis (Metamorfose)

A Metamorphosis é o processo pelo qual uma criatura se transforma permanentemente noutra espécie. É diferente da evolução normal porque é uma mudança radical e muitas vezes irreversível. Regras:

· Certas criaturas possuem um "gatilho de metamorfose" definido nos seus genes: ao atingir um determinado nível de experiência, ao morrer, ao ser infectada, ou ao entrar num bioma específico.
· Exemplos:
· Uma Flower Bud pode metamorfosear-se em Lulliar ao atingir o nível máximo.
· Um Crystal Sword pode tornar-se um Crystal Golem quando exposto a magia.
· Um humano infectado por MUSH transforma-se lentamente numa criatura MUSH.
· Algumas metamorfoses são cíclicas (ex.: licantropia com a lua cheia).
· A metamorfose é controlada por um sistema de "triggers" que compara o estado atual da unidade com as condições programadas. Quando todas as condições são verdadeiras, a substituição do tipo de criatura acontece.

Maps (Mapas)

A página Maps da wiki refere-se ao sistema de dimensionamento e visualização do mundo. Em termos de lógica, inclui:

· O mundo é dividido em chunks (pedaços) de 16×16 tiles para processamento eficiente.
· O mapa pode ser visualizado em vários níveis de zoom, sendo que o jogo renderiza apenas as chunks visíveis.
· Existe um "mapa do mundo" em miniatura que mostra uma versão simplificada do terreno e das fronteiras dos reinos.
· O jogador pode exportar e importar mapas através do Steam Workshop ou ficheiros locais. A estrutura de um ficheiro de mapa inclui camadas de tiles, biomas, recursos e posições iniciais das criaturas.

WorldBox – Simulador de Mundos em Camadas


Este documento descreve a lógica completa de WorldBox – God Simulator na sua versão atual (0.51.2). O jogo é um sandbox de simulação divina em que o jogador cria terrenos, popula o mundo com criaturas e observa civilizações emergirem, guerrearem e prosperarem, podendo intervir a qualquer momento. A simulação é organizada em camadas interdependentes que, juntas, geram um ecossistema vivo e autónomo.
WorldBox: A Enciclopédia Completa do Simulador de Mundo

Visão Geral e Filosofia do Jogo

WorldBox é um simulador de mundo em camadas na forma de um sandbox/god simulator, onde o jogador assume o papel de uma divindade capaz de criar, moldar e destruir mundos inteiros. Desenvolvido por Maxim Karpenko e lançado em 2012, o jogo opera na versão 0.51.2 e permite que o jogador crie o mapa, defina leis e poderes, povoe o mundo com criaturas e observe civilizações inteiras emergirem, evoluírem, guerrearem e deixarem seus próprios registros históricos.

O núcleo do jogo é um ciclo de observação e intervenção. O jogador gera o mundo, coloca vida, acompanha animais e povos, e depois vê essas populações evoluírem em vilas, reinos, alianças e guerras. A página oficial descreve o jogo como um simulador em que você cria ou destrói mundos, permitindo que civilizações cresçam, colonizem, naveguem e entrem em guerra. A lógica central é um ecossistema em camadas, onde cada sistema interage com os outros para criar uma simulação viva e dinâmica.

Estrutura da Interface e Visão do Jogador

O jogo se organiza visualmente em torno de uma barra de poderes e menus de inspeção. Existem cerca de 374 poderes organizados em 8 abas principais: Main, Unit, World Shaping, Noosphere and Life, Animals/Creatures/Monsters, Nature and Disasters, Destruction and Chaos, e Other various powers.

O menu principal aparece ao abrir o jogo ou ao sair das outras abas. Quase tudo que importa tem um menu próprio: reinos, cidades, cultura, clãs, exércitos, histórico do mundo, leis do mundo, idades e conhecimento. A interface foi projetada para que o jogador possa acessar rapidamente qualquer aspecto da simulação e intervir quando desejar.

O sistema de Knowledge é um arquivo pessoal de desbloqueios, listando 774 itens desbloqueáveis, divididos entre unidades, itens, genes, traits de criaturas, traits de subespécies, cultura, linguagem e clãs. As conquistas (achievements), em número de 96 na versão desktop e 86 no mobile, também fazem parte da progressão do jogador.

Primeira Camada: O Mundo e Sua Simulação Base

3.1 Mapa e Terreno

O terreno é a base sobre a qual tudo se constrói. O jogo opera com um sistema de tiles, onde cada tile pode conter um tipo de terreno (grama, areia, floresta, montanha, águas rasas, oceano, etc.). Esses tiles são agrupados para formar biomas funcionais.

3.2 Biomas: As Variações Funcionais do Terreno

Os biomas são variações funcionais do terreno, colocadas pelas sementes na aba "Nature and Disasters". Existem 23 biomas normais e 29 contando os especiais. Cada bioma altera a fauna, flora, traits das criaturas, cores dos bichos, crescimento de recursos, geração de loot e até quais criaturas podem nascer ali.

Mecânica de Biomas

Cada bioma tem força de crescimento, chance de geração, tamanho máximo e regras de espalhamento. A velocidade de espalhamento muda conforme a idade do mundo. Biomas com força 6 se expandem mais rápido que os de força 5, e biomas especiais como o Corrupted têm regras próprias de crescimento durante eras específicas.

O Corrupted Biome merece destaque: ele cresce mais rápido na Age of Dark and Despair, pode aplicar o efeito Cursed nas criaturas, transformar mortos em esqueletos e até gerar fantasmas. É a única fonte natural de osso em bioma, sendo extremamente perigoso para civilizações.

3.3 Leis do Mundo: As Regras da Física

As World Laws são os botões que mudam as regras fundamentais do mundo. Elas definem o que a simulação pode ou não pode fazer. Exemplos de leis incluem:

· 100 People: Limita a população por assentamento.
· Gaia's Covenant: Impede que fogo, grey goo, heat rays, antimatter bombs, granadas, meteoritos, bowling balls e Conway's Game of Life destruam o terreno/biomas.
· Diplomacy: Libera guerras, alianças, paz e rebeliões.
· Magic Rites: Permite rituais ligados à religião do clã.
· Mutant Box: Quando ativada, aplica de 1 a 4 traits aleatórios em criaturas ao nascerem.

As leis também controlam aspectos como fome, envelhecimento, desastres naturais e outros eventos.

3.4 Eras e Idades

As Ages são eras que alteram o mundo e duram pelo menos 30 anos. Existem 10 idades diferentes, e o jogador pode controlar sua sequência pelo Age Clock. As idades afetam o comportamento dos biomas, a taxa de crescimento, o surgimento de criaturas e outros aspectos da simulação ao longo do tempo, mudando o "clima" da simulação.

3.5 História do Mundo

O World History registra automaticamente eventos para o jogador acompanhar a narrativa do mundo em tempo real ou depois. A história pode ser filtrada por reis, unidades favoritas, cidades, reinos, guerras, clãs e desastres, permitindo que o jogador acompanhe a evolução de seu mundo ao longo do tempo.

Segunda Camada: Unidades e Genética

4.1 Unidades: A Definição Base

"Units" é o termo-base para todas as entidades com menu próprio: criaturas, pessoas, barcos, estruturas móveis e outras entidades interagíveis. Cada unidade possui stats que são o motor da simulação, determinando sua eficácia em combate, trabalho e sobrevivência.

4.2 Stats das Unidades

Os stats são números que definem a eficácia de cada unidade. Eles podem ser muito afetados por traits, idade e cultura. Para subespécies, os valores aparecem comparados ao genoma base, com cores indicando se estão acima (verde), iguais (amarelo) ou abaixo (vermelho).

Stats principais incluem:

· Saúde (Health)
· Ataque (Attack)
· Velocidade (Speed)
· Felicidade (Happiness): varia de -100 a 100
· Fome (Hunger): cai 1 ponto a cada 1,6 meses; quando chega a 0, a unidade começa a perder vida por fome
· Nutrição (Nutrition): restaurada pela comida
· Stamina: usada em ações de combate
· Outros stats ocultos: racionalidade, supply timer modifier, dodge, etc.

4.3 Genética e Traits

O sistema genético é uma das bases mais fundamentais do comportamento das espécies. A wiki oficial trata a genética como um sistema central, não apenas como detalhe de lore.

Como os Genes Funcionam

Cada forma de vida possui um ou mais cromossomos, e cada cromossomo contém diversos genes dispostos em "assentos" (gene seats). Esses genes afetam diretamente os stats da criatura, como força física, poder de ataque e velocidade. O efeito pode ser amplificado ou diminuído de acordo com a interação entre genes adjacentes.

Estados dos Genes

Cada assento de gene pode ter três estados:

· Dourado (Golden/Synergized): Dobra o bônus original do gene, fornecendo um enorme aumento de stats. Para converter um gene normal em dourado, todos os lados/nós do gene devem estar em sinergia.
· Normal: Estado padrão, sem modificadores.
· Ruim (Bad): Reduz o bônus do gene pela metade, criado quando um gene prejudicial ou amplificador prejudicial está adjacente ao gene.

Sinergia e Amplificadores

A sinergia ocorre quando as cores dos nós de genes adjacentes coincidem. Amplificadores dourados são posições genéticas especiais que podem criar sinergia com qualquer gene. Genes mutagênicos podem obter efeitos similares aos amplificadores.

Traits de Criaturas

Traits são características que modificam stats ou dão habilidades especiais. Eles são classificados entre personalidade e genética no código, alguns dados no nascimento, outros herdados e outros adquiridos aleatoriamente.

Criaturas civilizadas podem ganhar traits ao nascer ou interagindo com o mundo (como através de comida). O jogador também pode dar traits manualmente pelo editor de traits, quando desbloqueado. Traits são desbloqueados passando o mouse sobre itens novos (indicados por um círculo azul) ou completando achievements.

Traits de Subespécie

Existem 204 traits de subespécie para desbloquear. Eles têm raridades:

· Normal: +10 de mutação se estiver no pool de mutação
· Rare: +6 de mutação
· Epic: +3 de mutação
· Legendary: desbloqueado com achievement, +1 de mutação

Se a lei mundial "Mutant Box" estiver ativada, de 1 a 4 traits aleatórios são aplicados a criaturas ao nascerem. Traits que não podem aparecer aleatoriamente incluem: Grin Mark, Antimatter Essence, Geophagy, Unmoving, GMO, Uplifted e fenótipos.

Herança e Reprodução

Traits podem ser herdados. Se não houver chance de herança especificada, ela é calculada automaticamente como: Chance de Nascimento × 10. Por exemplo, uma chance de nascimento de 1.5% resulta em 15% de chance de herança.

A reprodução é definida por traits de subespécie e pode ser sexuada, assexuada ou por ataque. Existe oviparidade e viviparidade, e o birth rate dá tentativas extras de gerar filhotes. A reprodução sexuada requer parceiros, enquanto a assexuada permite que uma única unidade gere descendentes.

4.4 Phenotypes e Aparência

Os fenótipos determinam a aparência visual das criaturas, incluindo cor da pele, cabelo e outras características visuais. Eles são influenciados pelo bioma onde a criatura nasceu e por traits genéticos. A paleta de cores da maioria das criaturas muda dependendo do bioma onde elas surgiram.

4.6 Categorias Políticas das Criaturas

O jogo divide criaturas em "reinos" ocultos de amizade/hostilidade. Os civilizados clássicos ficam separados de forma explícita, e os animais evoluídos também têm blocos próprios. Essa camada determina quem é amigo, inimigo, civ, bandit, nature creature, neutral ou parte de categorias como snake, crab, dwarf e outras.

4.7 Doenças e Infecções

O jogo trata várias "doenças" como estados que enfraquecem e transformam criaturas:

· MUSH Spores: Só pode ser introduzido pelo jogador. Espalha em 70% dos casos quando uma criatura infectada morre num raio de 3 tiles. Pode ser curado por Divine Light, cure spell e accelerated healing.
· Zombie Infection: Transforma criaturas em zumbis.
· Plague: Praga que se espalha entre criaturas.

4.8 Estados e Efeitos

Status Effects são efeitos temporários que modificam as capacidades das unidades. Eles podem ser positivos (buffs) ou negativos (debuffs), e ao contrário dos traits, têm tempo limitado. Os principais estados incluem:

· Silence: Impede o uso de magias.
· Shield: Concede proteção temporária.
· Curse: Amaldiçoa a unidade.
· Madness: Causa loucura.
· Blessed: Abençoado.
· Cursed: Amaldiçoado.
· Shielded: Protegido.
· Frozen: Congelado.
· Burning: Em chamas.

4.9 Fome e Felicidade

A fome é um sistema fundamental: cada unidade perde 1 ponto de fome a cada 1,6 meses. Quando a fome chega a 0, a unidade começa a perder vida. A comida restaura fome/nutrição e saúde. A felicidade varia de -100 a 100 e afeta o comportamento da unidade, incluindo produtividade e propensão a se rebelar.

Terceira Camada: Civilização e Sociedade

5.1 Vilas e Cidades

As vilas são as regiões habitadas por unidades civilizadas. Nelas, os moradores constroem casas e estruturas, reproduzem, coletam comida, montam exércitos e podem mudar de reino durante guerras.

Construção de Cidades

Cada cidade tem um cooldown fixo de 1 mês para planejar novos prédios. Não pode planejar construção enquanto houver inimigos na área, incêndio ou captura em andamento. Só pode construir ou melhorar um prédio por vez. A escolha do que construir depende de tecnologia, população, limites de construção e pré-requisitos de outros prédios.

Visualmente, as cidades são moldadas pelo comportamento da raça:

· Humanos, elfos e orcs escolhem localizações de forma mais aleatória.
· Anões tendem a construir mais no centro, deixando vilas mais "organizadas".

Estruturas da Cidade

· Casas: Prédios onde unidades sapientes entram. Os quatro civs clássicos têm temas visuais e várias tiers de casa, com duas variações aleatórias por tier acima do nível 0. Raças não-clássicas usam arquitetura de uma única tier, equivalente à casa tier 1 dos clássicos.
· Town Hall: Centro administrativo, guarda até 3 livros.
· Library: Biblioteca, guarda até 5 livros.
· Market: Mercado para comércio.
· Mines: Minas para extração de recursos.
· Barracks: Quartéis para treinamento militar.
· Docks: Ficam na costa, têm dois níveis. Liberam a construção de barcos, funcionam como parada de barcos de comércio e ponto de retorno/reparo, e aumentam o máximo de casas da vila em +1.

5.2 Trabalho e Tasks

Em civilizações, cada cidadão pode ter um job, e o job determina a maior parte das behaviour tasks. A wiki separa 117 behaviour tasks diferentes. Os jobs incluem:

· Coletor de comida
· Construtor
· Mineiro
· Soldado
· Líder de vila
· Comerciante
· Outros trabalhos especializados

5.3 Economia e Recursos

A economia em WorldBox é um sistema amplo que cobre gold, loot, money/coins, impostos, tributos e compartilhamento de recursos, expandido na versão 0.50.0.

Recursos Principais

· Madeira: Obtida de árvores, usada para construção.
· Comida: Obtida por coleta, agricultura e pesca.
· Stone: Pedra obtida por mineração.
· Common Metals: Metais comuns (cobre, bronze).
· Gems: Gemas preciosas.
· Gold: O único recurso monetário principal. Não nasce naturalmente em biomas; é obtido por minas e por barcos de comércio que geram ouro ao se moverem entre docks.
· Herb: Ervas medicinais.
· Bread: Pão produzido por civilizações.

Troca e Comércio

A troca entre cidades é específica: reinos não trocam entre si, apenas vilas do mesmo reino compartilham recursos, e isso só acontece depois que a tecnologia de trading é desbloqueada. Existe um supply timer com cooldown base de 6 meses e modificadores ligados ao líder da vila e a traits.

5.4 Barcos e Transporte

Os boats são o transporte entre massas de terra. Existem três tipos principais:

· Trading Boats: Geram 5 gold quando chegam a um dock de reino não inimigo.
· Fishing Boats: Produzem comida.
· Transport Boats: Movem exércitos e colonos.

Cada raça possui versões próprias de dois tipos de barcos. Apesar de serem veículos inanimados, eles têm menu de unidade e podem ser "mortos".

Os docks ficam na costa, têm dois níveis e permitem construir barcos. Também servem como parada de barcos de comércio e ponto de retorno/reparo, além de aumentar o máximo de casas da vila em +1.

5.5 Reinos e Política

Kingdoms são feitos de uma ou mais villages e são liderados por kings. Eles podem fazer guerra, formar alianças e influenciar cidadãos.

Kingdom Opinions

As relações entre reinos são calculadas por um sistema de opiniões. A opinião depende de vários fatores, como diplomacia do rei, humor do rei, distância das fronteiras, estar em guerra e até "inimigos comuns". Essa pontuação influencia quando guerras e alianças começam.

Alianças

As alianças são grupos de dois ou mais reinos criados para defesa mútua ou para enfrentar reinos mais fortes. Elas podem surgir por plot ou pela power Unity.

Guerra

A guerra é o confronto entre pelo menos dois reinos e normalmente envolve armys, levando à expansão de um lado e queda do outro. Cada vila tem seu próprio army, composto por cidadãos, essenciais para defender vilas e vencer guerras.

5.6 Clãs e Famílias

Os clãs são linhagens reais/políticas que geralmente sustentam poder em reinos e vilas. Apenas membros de clã podem executar vários plots importantes, como declarar guerra, formar alianças e escrever livros.

As famílias representam parentesco e são formadas por pais e filhos. O tab de genealogia é mais útil visualmente para rastrear linhagens completas.

WorldBox separa sangue, poder, herança e política como camadas diferentes da simulação. Clãs, famílias, genealogia e posição de liderança são sistemas distintos que interagem entre si.

5.7 Culturas e Idiomas

As culturas são um meta-sistema social/funcional. A cultura nasce nas bonfires, se espalha por zonas da mesma raça e pode existir independentemente de fronteiras de vila e reino. A cultura reúne nome, banner, traits, livros, stats e mais, sendo o que faz civilizações ficarem diferentes ao longo do tempo.

Os idiomas (Languages) são meta-objetos relacionados aos livros que as civilizações escrevem. Apenas membros de clã podem escrever livros via New Book plot, precisando de inteligência, warfare, stewardship e diplomacy mínimos, level 10, 200 money e cultura/idioma/vila com espaço de livro.

5.8 Livros e Conhecimento

Books são um recurso especial guardado pelas vilas que dá efeitos de stats quando lido. Eles são escritos por membros de clã e armazenados em Libraries (até 5) e Town Halls (até 3).

5.9 Plots e Conspirações

Os plots são uma camada de previsibilidade: eles mostram ao jogador o progresso de guerra, rebelião e outros eventos antes de acontecerem. Ap

Camada 1 – Base do Mundo: Terreno, Biomas e Recursos

Mapa e Terreno

O mundo é uma grelha de tiles. Cada tile pode conter água (oceano ou águas rasas), terra (areia, solo, rocha, neve, etc.) ou árvores e arbustos. O jogador constrói e modifica o terreno com os poderes do separador "Create and Shape Your World" e com as sementes e desastres do separador "Nature and Disasters".

Biomas

Um bioma é um conjunto de tiles com regras próprias. Cada bioma controla que árvores, plantas e animais aparecem, que bónus ou efeitos as criaturas recebem e até a paleta de cores das unidades. Existem 23 biomas normais e 29 se contarmos os especiais (Sand, Hill e os biomas‑spawner: Tumor, Biomass, Pumpkin e Cyber). Os biomas espalham‑se automaticamente: a cada 0,01‑0,02 meses o jogo atualiza a vizinhança de cada tile e converte os tiles adjacentes de acordo com a força e as regras do bioma. A velocidade de expansão também depende da Idade do mundo atual.

Biomas de força 6 (expandem‑se mais depressa): Arcane Desert, Candy, Corrupted, Crystal, Enchanted, Infernal, Jungle, Lemon, Mushroom, Permafrost, Savanna, Swamp, Wasteland.

Biomas de força 5: Birch, Celestial, Clover, Flower, Garlic, Grass, Maple, Paradox, Rocklands, Singularity.

· Especiais: Corrupted cresce ainda mais rápido durante a Age of Dark and Despair, pode amaldiçoar unidades, transforma mortos em esqueletos e gera fantasmas; Biomass e Cybercore comportam‑se como creep biomes – entidades estacionárias que expandem os seus tiles e geram unidades periodicamente.

Recursos Naturais

O terreno gera recursos que as civilizações recolhem e transformam.

· Madeira – Obtida de árvores.
· Pedra – Obtida de montanhas e colinas.
· Comida – Arbustos de frutos, animais caçados, peixes e culturas agrícolas.
· Metais comuns – Ferro, cobre, bronze (fundido a partir de minério).
· Metais preciosos – Prata, Mythril, Adamantine.
· Ouro – Único recurso monetário; obtido em minas e através de barcos de comércio.
· Gemas – Usadas em joalharia.
· Couro e Osso – Obtidos de animais abatidos.

Todos os recursos são armazenados nos armazéns da vila (stockpiles) e usados para construir edifícios, fabricar equipamento ou alimentar a população.

Camada 2 – Unidades: Tipos, Atributos e Genética

O que é uma Unidade

"Unit" é o termo genérico para qualquer entidade que tenha um menu próprio: criaturas, pessoas, barcos, algumas estruturas móveis. Cada unidade possui um conjunto de atributos (stats) que regem o seu comportamento.

Atributos de uma Unidade (Stats)

Atributo Descrição
Saúde (Health) Vida máxima. Quando chega a zero, a unidade morre.
Dano (Damage) Dano base por ataque.
Velocidade (Speed) Rapidez de movimento no mapa.
Esquiva (Dodge) Probabilidade de evitar um ataque.
Precisão (Accuracy) Probabilidade de acertar um ataque.
Armadura (Armor) Redução percentual do dano recebido.
Diplomacia (Diplomacy) Influencia a opinião entre reinos.
Felicidade (Happiness) Varia de -100 a 100; afeta produtividade e risco de rebelião.
Fome (Hunger) Desce 1 ponto a cada 1,6 meses. Ao atingir 0, a unidade perde vida por inanição.

Os atributos são afetados por traits, idade, cultura, equipamento e estado de saúde.

Traits (Características)

Traits são modificadores temporários ou permanentes que alteram os atributos e o comportamento. Existem mais de 200 traits, divididos em sete categorias principais: Body, Mind, Spirit, Acquired, Fun, Miscellaneous e Special.

· Hereditariedade: traits indicam se podem ser herdados; se não houver percentagem explícita, a probabilidade de herança é calculada como Chance de Nascimento × 10.
· Atribuição: traits são dados ao nascer (pela genética ou aleatoriamente), ao interagir com o mundo (comer certos alimentos, sofrer uma maldição) ou pelo jogador através do editor de traits.
· Efeitos especiais: alguns traits concedem habilidades ativas (voar, nadar, curar‑se) ou impõem restrições (não poder reproduzir‑se, não poder falar).

Genética e Reprodução

Cada espécie tem um genome base. As subespécies variam desse genome e os seus valores são comparados com o base, surgindo a verde (acima), amarelo (igual) ou vermelho (abaixo). O jogador pode editar genes com o Gene Editor.

A reprodução pode ser:

· Sexuada – Requer dois parceiros.
· Assexuada – Uma unidade gera descendentes sozinha.
· Por ataque – Algumas criaturas reproduzem‑se ao atacar (ex.: zombies, Grey Goo).

Parâmetros reprodutivos:

· Birth Rate – Número de tentativas de gerar filhos.
· Oviparidade / Viviparidade – Determinam se a unidade põe ovos ou dá à luz diretamente.
· Tempo de gestação – Período até ao nascimento.

Fenótipos

A aparência visual (cor da pele, cabelo, roupas) é determinada por fenótipos, que dependem do bioma onde a unidade nasceu e de traits genéticos.

Camada 3 – Comportamento: Tasks, Necessidades e Efeitos de Estado

Tasks (Tarefas)

Cada unidade executa tarefas (tasks) de forma contínua. As tarefas são selecionadas com base na profissão (job) e nos traits. Existem 117 tasks de comportamento registadas na wiki, agrupadas em categorias:

· Init (inicialização)
· Movement (movimento)
· Combat (combate)
· Reproduction (reprodução)
· Children (crianças)
· Socializing (socialização)
· Deliver Resources (entrega de recursos)
· Reflection (momentos de introspeção / lazer)

Exemplo – Sistema de Lazer (Reflection):

1. Se a subespécie tem o trait Super Positivity, executa tarefas alegres (rir, cantar, brincar).
2. Senão, o jogo avalia o estado da unidade e monta um Task Pool (conjunto de tarefas possíveis) com pesos:
   · Se feliz (>50), adiciona Happy Laughing, Singing.
   · Se infeliz (< -70), adiciona Swearing, Punch a Tree, Crying.
   · Se tem o trait pyromaniac, adiciona Start Fire.
   · Se é criança, adiciona brincadeiras infantis.
3. O jogo sorteia uma tarefa do Task Pool.

Necessidades Básicas

· Fome: diminui 1 ponto a cada 1,6 meses. A unidade procura comida quando está baixa.
· Saúde: diminui ao receber dano, sobe ao comer ou receber curas.
· Felicidade: afetada por ter emprego, casa, comida, ausência de guerra, etc.

Efeitos de Estado (Status Effects)

Efeitos temporários que modificam as capacidades da unidade. Diferem dos traits por terem duração limitada. Exemplos: Silence (impede magias), Shield (absorve dano), Curse (amaldiçoado), Madness (age aleatoriamente), Burning (perde vida por fogo), Frozen (imobilizado).

Doenças

Funcionam como efeitos de estado especiais que podem propagar‑se:

· MUSH Spores – Só pode ser introduzida pelo jogador. Quando uma unidade infetada morre, há 70% de hipótese de espalhar esporos num raio de 3 tiles.
· Zombie Infection – Transforma a vítima num zombie.
· Plague – Propaga‑se entre unidades próximas.

Doenças podem ser curadas com Divine Light, poções de cura ou habilidades como Accelerated Healing.

Camada 4 – Combate e Magia

Início do Combate

Quando duas unidades de reinos hostis (ou uma unidade hostil e uma pacífica) se aproximam, o jogo avalia se há um inimigo para atacar. A unidade escolhe entre ataque corpo a corpo, à distância ou magia, consoante o que possui e os estados ativos (Silence impede magia, por exemplo).

Tipos de Ataque

· Melee – Golpe corpo a corpo; consome stamina.
· Ranged – Dispara um projétil (flecha, pedra, etc.).
· Spell – Magia ofensiva ou utilitária.

Magias (Spells)

Algumas unidades (mages) atacam primariamente com magias. Ao atacar, tentam primeiro executar uma magia. Se a magia falhar (pela percentagem de sucesso), usam o ataque normal.

Magia Unidade Alvo Sucesso Distância Mínima Efeito
Lightning Evil Mage Inimigos 30% 6 Invoca um relâmpago médio no local.
Tornado Evil Mage Inimigos 2% 9 Invoca um tornado.
Fire Evil Mage Inimigos 20% 3 Cria fogo no local e em 3 tiles adjacentes.
Curse Necromancer Inimigos 20% 4 Amaldiçoa a área.
Random Teleport Evil/White Mage Próprio 70% (quando saúde < 60%) 6 Teletransporta‑se para terra firme aleatória.
Blood Rain Todos os mages Próprio 50% (quando saúde < 90%) 0 Cura a unidade com chuva de sangue.
Spawn Skeleton Necromancer Próprio 60% (se ≤6 esqueletos na chunk) 0 Invoca 1 esqueleto.
Shield White Mage Próprio 20% 0 Concede 30 segundos de escudo.
Cure Plague Doctor Próprio 20% 0 Lança gotas de cura sobre si mesmo.
Spawn Tree Fertilizer Druid Região 100% 0 Fertiliza árvores em redor.

Projéteis

Ao usar armas de longo alcance ou certas magias, o jogo gera projéteis. Cada um tem um efeito próprio:

Projétil Efeito
Arrow Dano perfurante.
Snowball Congela a unidade.
Rock Dano contundente.
Firebomb Incendeia a área.
Fireball Pequena explosão de fogo.
Madness Ball Causa loucura.
Plasma Ball Dano elétrico.
Shotgun Bullet Vários fragmentos.
Red/Blue/Green Orb Efeitos variados (fogo, gelo, lentidão).
Bone Dano físico; usado por necromancers.
Skull Projétil amaldiçoado.
Cannonball Destrói terreno e construções.
Acid Ball Dissolve unidades e tiles.

Morte e Pilhagem

Quando uma unidade morre, deixa cair o seu equipamento e recursos (loot). O corpo pode transformar‑se num esqueleto (se estiver num bioma Corrupted) ou num fantasma.

Camada 5 – Civilização: Vilas, Reinos e Política

Fundação de uma Vila

Quando uma criatura sapiente (com o trait Prefrontal Cortex) encontra um local adequado, funda uma vila. A primeira estrutura é a Bonfire, que cria uma nova cultura.

Construção de Edifícios

As vilas constroem automaticamente ao longo do tempo. A lógica de construção obedece a regras rígidas:

· A vila só pode planear um novo edifício a cada 1 mês.
· Não pode planear se houver inimigos na área, incêndio, ou se estiver a ser conquistada.
· Só constrói ou melhora um edifício de cada vez.
· A escolha do que construir depende da tecnologia disponível, população, limites de construção e pré‑requisitos de outros edifícios.

Lista de Edifícios Civilizacionais:

Edifício Função
Bonfire Funda a vila e a cultura; +3 de capacidade de casas.
House (Tent → Tier 1‑6) Abriga cidadãos. Cada tier suporta mais habitantes.
Town Hall Residência do rei/líder; guarda até 3 livros.
Well Decoração; +3 de capacidade de casas; pode ser usado para apagar fogos.
Windmill Cria terrenos agrícolas (fields) à sua volta.
Mine Permite que mineiros extraiam minerais; os minerais aparecem nos tiles adjacentes.
Temple +2 de capacidade de casas; local de oração.
Docks (Tier 1‑2) Constrói e estaciona barcos; +1 de capacidade de casas; serve de ponto de comércio.
Watchtower Dispara 3 flechas de cada vez contra inimigos.
Barracks Treina e mantém o exército.
Library Guarda até 5 livros.
Market Ponto de troca de recursos.

Economia e Comércio

· Ouro: único recurso monetário. Usado para construir edifícios, recrutar exércitos e manter a vila. Obtido por minas e por barcos de comércio que geram 5 de ouro ao chegar a um dock de um reino não inimigo.
· Troca entre vilas: só acontece entre vilas do mesmo reino e apenas depois de desbloqueada a tecnologia Trading. A troca é efetuada por um temporizador (supply timer) com cooldown base de 6 meses, podendo ser modificado pelas características do líder.

Reinos e Diplomacia

· Reino: conjunto de uma ou mais vilas sob o comando de um rei. Cada reino tem nome, bandeira, lema e atributos.
· Kingdom Opinions: sistema de pontuação que determina a atitude de um reino em relação a outro. Fatores: diplomacia do rei, humor do rei, distância das fronteiras, guerras em curso, inimigos comuns. A pontuação decide se os reinos vão formar alianças ou declarar guerra.
· Alianças: grupos de 2 ou mais reinos para defesa mútua. Podem surgir por diplomacia natural ou por intervenção do jogador (power Unity).
· Guerra: confronto armado entre reinos. As vilas produzem exércitos e lutam até que um lado seja destruído ou se faça a paz.

Líderes e Plots

· Kings e Líderes de Vila: governam e tomam decisões. Só membros de clãs podem executar plots importantes.
· Plots: planos de ação (declarar guerra, rebelar‑se, escrever um livro, etc.) que aparecem como ícones no mapa. O jogador pode ver o progresso, cancelar ou forçar o plot.

Clãs e Famílias

· Famílias: relações de parentesco baseadas em pais e filhos. O separador de Genealogia mostra a árvore completa.
· Clãs: linhagens reais/políticas que detêm o poder. Um clã pode ter traits de clã que afetam a lealdade e a influência.

Culturas e Línguas

· Cultura: nasce na Bonfire e propaga‑se por zonas da mesma raça. Define o nome da cultura, bandeira, traits, livros e aparência.
· Língua: associada aos livros escritos pela civilização. Línguas com o trait Melodic influenciam as tarefas de lazer (aumentam a probabilidade de cantar).

Camada 7 – Barcos, Equipamento e Itens Especiais

Barcos

Unidades especiais que transportam tropas, pescam ou fazem comércio. Existem três tipos principais:

Tipo Função
Transport Move exércitos e colonos.
Fishing Produz comida.
Trading Navega até docks de reinos não inimigos e gera 5 de ouro ao chegar.

Os barcos são construídos nos Docks e podem ser destruídos como qualquer unidade.

Equipamento

Armas e armaduras são fabricadas pelas civilizações com base nos recursos disponíveis. Classificam‑se em:

· Armas corpo a corpo: Sword, Axe, Hammer, Spear.
· Armas de longo alcance: Bow.
· Armaduras: Helmet, Armor, Boots.
· Acessórios: Ring, Amulet.

Os materiais seguem uma hierarquia: Wood, Stone, Copper, Bronze, Silver, Iron, Steel, Mythril, Adamantine. Couro e osso são usados para armaduras e acessórios.

Raridade: determinada pelo modifier level do item (Normal → Rare → Epic → Legendary). Itens lendários recebem um nome único.

Armas especiais: Flame Sword (dropped por Demons), Evil Staff, White Staff, Druid Staff, Necromancer Staff, Plague Doctor Staff, Alien Blaster, Shotgun.

Camada 8 – Linha do Tempo: Idades e História do Mundo

Idades (Ages)

O mundo atravessa 10 idades diferentes. Cada idade dura no mínimo 30 anos e afeta o comportamento dos biomas, a taxa de crescimento das plantas, o aparecimento de desastres e a agressividade das criaturas. O jogador pode controlar a sequência através do Age Clock.

História do Mundo (World History)

Regista automaticamente todos os eventos importantes: fundações de reinos, guerras, tratados de paz, desastres naturais, mortes de reis. A história pode ser filtrada por reis, unidades favoritas, cidades, reinos, guerras, clãs e desastres. O jogador também pode adicionar as suas próprias notas através do World History Marker.

Wiki essencial de WorldBox

1) O que o jogo é, na prática

O núcleo do jogo é um ciclo de observação e intervenção. Você gera o mundo, coloca vida, acompanha animais e povos, e depois vê essas populações evoluírem em vilas, reinos, alianças e guerras. A página oficial/Steam reforça que o jogo é sobre criar o mundo, preencher com criaturas e assistir as civilizações crescerem, colonizarem novas terras e irem à guerra. 

2) Interface e “visão do jogador”

Visualmente, o jogo gira em torno da barra de poderes e menus de inspeção. A wiki oficial diz que existem cerca de 374 poderes organizados em 8 abas: Main, Unit, World Shaping, Noosphere and Life, Animals/Creatures/Monsters, Nature and Disasters, Destruction and Chaos, e Other various powers. O menu principal aparece ao abrir o jogo ou ao sair das outras abas. 

Na lógica visual, quase tudo que importa tem um menu próprio: reinos, cidades, cultura, clãs, exércitos, histórico do mundo, leis do mundo, idades e conhecimento. A própria página inicial da wiki destaca essas áreas como páginas centrais do sistema. 

3) Mapa, terreno e biomas

Os biomas são variações funcionais do terreno. A wiki registra 23 biomas distintos e 29 se contar os especiais. Eles mudam o que nasce ali, o que as criaturas recebem de bônus/efeitos e até a paleta visual delas. Alguns biomas só existem em um lugar específico e alguns mobs só surgem em biomas específicos. 

Os biomas especiais mais marcantes têm comportamento próprio. O Corrupted Biome cresce mais rápido na Age of Dark and Despair, pode aplicar Cursed nas criaturas, transformar mortos em esqueletos e até gerar fantasmas; ele também é a única fonte natural de osso em bioma, sendo muito perigoso para civilizações. Já Biomass e Cybercore são “creep biomes”: entidades estacionárias que espalham seus tiles e geram unidades periodicamente. 

4) Leis do mundo

As World Laws são os botões que mudam a física e as regras do mapa. Exemplos citados na wiki: fogo e grey goo não se espalham, heat rays não viram lava/aqueçam lava existente, bombas antimatter não destroem terreno, meteoritos/granadas/bowling balls não removem biomas e civs não conseguem usar Cast Grass Seeds. Em outras palavras, as leis definem o que a simulação pode ou não pode fazer. 

5) Criaturas, unidades, stats e traits

Toda criatura pode ter traits, e elas mudam estatísticas, aparência e comportamento. A wiki diz que criaturas civilizadas podem ganhar traits ao nascer ou interagindo com o mundo, e o jogador também pode dar traits manualmente pelo editor, quando estiver desbloqueado. A própria wiki mostra que traits podem ser obtidos por hover em itens novos ou por achievements. 

Os stats são números que definem a eficácia de cada unidade. Eles podem ser muito afetados por traits, idade e cultura. A lógica é simples: quanto maior o stat, melhor o desempenho naquela função. Para subspecies, os valores aparecem comparados ao genome base, com cores indicando se estão acima, iguais ou abaixo. 

Também existe uma camada de progressão de descoberta. O menu de Knowledge é um arquivo pessoal de desbloqueios; a wiki lista 774 unlockables, divididos entre unidades, itens, genes, traits de criaturas, traits de subspecies, cultura, linguagem e clãs. 

6) Família, clã e linhagem

As famílias representam parentesco e são formadas por pais e filhos; a wiki observa que elas nem sempre refletem perfeitamente a genealogia completa, então o tab de genealogia costuma ser mais útil visualmente. Já os clãs são linhagens reais/políticas: geralmente sustentam poder em reinos e vilas, e só membros de clã podem executar vários plots importantes. 

Em termos visuais e lógicos, isso significa que WorldBox não trata civilização só como “população”; ele separa sangue, poder, herança e política. Clãs, famílias, genealogia e posição de liderança são camadas diferentes da simulação. 

7) Vilas, reinos e culturas

As vilas são as regiões habitadas por unidades civilizadas: nelas os moradores constroem casas e estruturas, reproduzem, coletam comida, montam exércitos e podem mudar de reino durante guerras. Um reino é composto por uma ou mais vilas e é liderado por um rei; ele pode fazer guerra, formar alianças e influenciar cidades e cidadãos. 

As culturas são um meta-sistema social/fucional. A cultura nasce nas bonfires, se espalha por zonas da mesma raça e pode existir independentemente de fronteiras de vila e reino. A wiki também diz que cultura reúne nome, banner, traits, livros, stats e mais. Em termos de jogo, cultura é o que faz civilizações ficarem diferentes no tempo. 

8) Construção de cidades

A página de City Building descreve a lógica de construção das cidades: cada cidade tem um cooldown fixo de 1 mês para planejar novos prédios, não pode planejar construção enquanto houver inimigos na área, incêndio ou captura em andamento, e só pode construir/upgradear um prédio por vez. A escolha do que construir depende de tecnologia, população, limites de construção e pré-requisitos de outros prédios. 

Visualmente, as cidades são moldadas pelo comportamento da raça. Humanos, elfos e orcs escolhem localizações de forma mais aleatória, enquanto anões tendem a construir mais no centro, deixando vilas mais “organizadas”. A própria página de construção cita esse padrão. 

9) Casas, docks, barcos e movimento

As casas são prédios que unidades sapientes entram; os quatro civs clássicos têm temas visuais e várias tiers de casa, com duas variações aleatórias por tier acima do nível 0. Raças não-clássicas usam uma arquitetura de uma única tier, equivalente à casa tier 1 dos clássicos. 

Os docks ficam na costa, têm dois níveis e liberam a construção de barcos. Eles também funcionam como parada de barcos de comércio e ponto de retorno/reparo, além de aumentarem o máximo de casas da vila em +1. Já os boats são o transporte entre massas de terra; existem três tipos principais, cada raça possui versões próprias de dois deles, e eles até têm menu de unidade e podem ser “mortos” apesar de serem veículos inanimados. 

10) Economia, ouro, recursos e troca

A economia em WorldBox é mais ampla do que só ouro: ela cobre gold, loot, money/coins, impostos, tributos e compartilhamento de recursos. A wiki define a economia como um sistema grande, ainda não totalmente desenvolvido, que foi ampliado na versão 0.50.0. 

O gold é o único recurso/currency metálico principal. Ele não nasce naturalmente em biomas; é obtido por minas e por algumas rotas econômicas, como barcos de comércio que geram ouro ao se moverem entre docks. O ouro é usado para manter e recrutar exércitos e para construir vários prédios. 

A troca entre cidades é bem específica: reinos não trocam entre si, apenas vilas do mesmo reino compartilham recursos, e isso só acontece depois que a tecnologia de trading é desbloqueada. Além disso, existe um supply timer com cooldown base de 6 meses e modificadores ligados ao líder da vila e a traits. 

11) Diplomacia, opiniões, alianças, guerra e planos

As relações entre reinos são calculadas por um sistema de Kingdom Opinions. A opinião depende de vários fatores, como diplomacia do rei, humor do rei, distância das fronteiras, estar em guerra e até “inimigos comuns”. Essa pontuação influencia quando guerras e alianças começam. 

As alianças são grupos de dois ou mais reinos criados para defesa mútua ou para enfrentar reinos mais fortes; elas podem surgir por plot ou pela power Unity. A guerra é o confronto entre pelo menos dois reinos e normalmente envolve armys, levando a expansão de um lado e queda do outro. 

Os plots servem como uma camada de previsibilidade: eles mostram ao jogador o progresso de guerra, rebelião e outros eventos antes de acontecerem. A wiki diz que os plots aparecem acima de unidades especiais e menus, geralmente são feitos por membros de clã e podem ser vistos, desbloqueados, forçados ou cancelados. 

12) Exércitos

Cada vila tem seu próprio army, e os exércitos são feitos a partir dos cidadãos da vila. Eles são essenciais para defender vilas e vencer guerras. A própria página de Armies da wiki destaca esse papel central, embora ela mesma esteja marcada com aviso de revisão. 

13) Idades, história do mundo e ritmo da simulação

As Ages são eras que alteram o mundo e duram pelo menos 30 anos; a wiki lista 10 idades e diz que o jogador pode controlar sua sequência pelo Age Clock. Isso é uma das formas mais importantes de mudar o “clima” da simulação ao longo do tempo. 

O World History registra eventos automaticamente para o jogador acompanhar a narrativa do mundo em tempo real ou depois. A história pode ser filtrada por reis, unidades favoritas, cidades, reinos, guerras, clãs e desastres. 

14) Achievements e desbloqueios

WorldBox tem 96 achievements na versão desktop e 86 no mobile, e muitos desbloqueios de conhecimento/traits dependem de jogar e descobrir coisas no próprio mundo. Isso faz o jogo ter uma camada de progressão além da simulação em si. 

15) O que mudou nas versões mais recentes

A página oficial de changelog diz que a versão atual em Android, iOS, Steam, macOS e Linux é 0.51.2. Nos lançamentos de setembro de 2025, a 0.51.0/0.51.1/0.51.2 trouxe melhorias de tradução e performance, correções de carregamento e novos conteúdos como efeitos/traits ligados a alienígenas, além de ajustes em menus e bugs.

Em uma frase: WorldBox funciona como um ecossistema de simulação em camadas, onde o jogador governa o terreno, as leis, os biomas, as criaturas e as civilizações; estas, por sua vez, constroem cidades, geram cultura, acumulam recursos, formam clãs, declaram guerras, fazem alianças e deixam registros históricos próprios.

Lógica central do jogo. WorldBox é um sandbox de deus em que você cria terreno, define leis do mundo, solta vida, acompanha a formação de ecossistemas e deixa criaturas virarem civilizações, reinos e guerras; o jogo foi descrito oficialmente como um simulador em que você cria ou destrói mundos e vê civilizações crescerem, colonizarem, navegarem e entrarem em guerra. 

Mundo, terreno e biomas. Os biomas são variações funcionais do terreno colocadas pelas sementes da aba “Nature and Disasters”; existem 23 biomas normais e 29 contando os especiais. A lista oficial inclui Arcane Desert, Birch, Candy, Celestial, Clover, Corrupted, Crystal, Enchanted, Flower, Garlic, Grass, Infernal, Jungle, Lemon, Maple, Mushroom, Paradox, Permafrost, Rocklands, Savanna, Singularity Swamp, Swamp e Wasteland; entre os especiais estão Sand, Hill e os biomas-spawner Tumor, Biomass, Pumpkin e Cyber. Biomas alteram fauna, traits, cores dos bichos, crescimento, geração de loot e até quais criaturas podem nascer ali. 

Como biomas realmente funcionam. Cada bioma tem força de crescimento, chance de geração, tamanho máximo e regras de espalhamento. A página oficial explica que biomas com força 6 incluem Arcane Desert, Candy, Corrupted, Crystal, Enchanted, Infernal, Jungle, Lemon, Mushroom, Permafrost, Savanna, Swamp e Wasteland, enquanto os de força 5 incluem Birch, Celestial, Clover, Flower, Garlic, Grass, Maple, Paradox, Rocklands e Singularity; a velocidade de espalhamento também muda conforme a idade do mundo. 

Leis do mundo, eras e história. As World Laws mudam as regras físicas do mundo: por exemplo, 100 People limita população por assentamento, Gaia’s Covenant impede fogo, grey goo, heat rays, antimatter bombs, granadas, meteoritos, bowling balls e Conway’s Game of Life de destruírem o terreno/biomas, Diplomacy libera guerras, alianças, paz e rebeliões, e Magic Rites permite rituais ligados à religião do clã. O sistema de eras também altera o comportamento do mapa, e o World History registra automaticamente os eventos do mundo para consulta posterior. 

Powers. O jogo tem cerca de 374 poderes em 8 abas: Main, Unit, World Shaping, Noosphere and Life, Animals/Creatures/Monsters, Nature and Disasters, Destruction and Chaos e Other various powers. Entre os poderes de informação e controle, a wiki destaca World Laws, Ages, World History, Eye of Insight e Divine Light; Divine Light remove doenças, machuca certas criaturas, encerra plots e interrompe tasks. 

Unidades, stats, fome, felicidade e reprodução. “Units” é o termo-base para criaturas, pessoas, barcos e outras entidades com menu próprio. Os stats da unidade são o motor da simulação; felicidade vai de -100 a 100; comida restaura fome/nutrição e saúde; a fome cai 1 a cada 1,6 meses, e ao chegar a 0 a unidade começa a perder vida por fome. A reprodução é definida por traits de subspecie e pode ser sexuada, assexuada ou por ataque; existe oviparidade e viviparidade, e o birth rate dá tentativas extras de gerar filhotes. 

Doenças e infecções. O jogo trata várias “doenças” como estados que enfraquecem e transformam criaturas. Entre as mais importantes estão MUSH Spores, zombie infection e plague; no caso do MUSH, a wiki diz que ele só pode ser introduzido pelo jogador, espalha em 70% dos casos quando uma criatura infectada morre num raio de 3 tiles e pode ser curado por Divine Light, cure spell e accelerated healing. 

Combate e spells. Quando uma unidade ataca, o jogo decide entre melee, ranged ou spell, dependendo do que ela possui e de estados como Silence, Spell Recovery, Slow e Combat Action Recovery. Spells têm prioridade parcial: se a unidade pode usar magia, ela pode atacar com spell, mas o jogo escolhe entre as opções válidas; ações de combate avançadas usam uma chance-base de combate e consomem stamina. 

Cidades, jobs, tasks e construção. Em civilizações, cada cidadão pode ter um job, e o job determina a maior parte das behaviour tasks. A wiki separa 117 behaviour tasks, e a construção de cidades tem regras rígidas: cada cidade planeja novos prédios a cada 1 mês, não planeja se houver inimigos, incêndio ou captura em andamento, e só constrói ou melhora um prédio por vez. O sistema de casas, town halls, libraries, markets, mines e docks faz parte da expansão da cidade. 

Economia e recursos. Resources são os itens usados para fabricar coisas, construir estruturas ou servir de comida, e as vilas os guardam em stockpiles. A wiki detalha madeira, comida, stone, common metals, gems, gold, herb, bread e muitos outros; food e materials têm origens ligadas a biomas, árvores, mineração, caça e produção civilizada. O ouro é o recurso monetário principal para várias construções e custos, mas a wiki também distingue money de gold no sistema de equipamentos. 

Books, culturas e idiomas. Books são um recurso especial guardado pelas vilas que dá efeitos de stats quando lido; apenas membros de clã podem escrever livros via New Book plot, precisando de inteligência, warfare, stewardship e diplomacy mínimos, level 10, 200 money e cultura/idioma/vila com espaço de livro. Libraries guardam até 5 livros e Town Halls até 3. Cultures e Languages são meta-objetos diferentes: cultura nasce nas bonfires e se espalha por zonas da mesma raça; linguagem se relaciona aos livros que as civilizações escrevem. 

Reinos, alianças, clãs e política. Kingdoms são feitos de uma ou mais villages e são liderados por kings; eles podem fazer guerra, formar alianças e influenciar cidadãos. O sistema de Kingdom Opinions calcula relações que afetam guerra e alianças, e Plots mostra antecipadamente o progresso de guerra, rebelião e outros planos; os líderes que fazem plots são membros de clã, normalmente village leaders ou kings. 

Boats e docks. Docks ficam nas costas, têm dois tiers e permitem construir boats; também servem como parada de barcos de comércio e retorno/reparo, além de dar +1 max houses. Boats têm três tipos principais; a wiki explica que boats de trading geram 5 gold quando chegam a um dock de reino não inimigo, os fishing boats produzem comida e os transport boats movem exércitos e colonos. 

Equipamentos e itens. O catálogo de Equipment se organiza em melee weapons (sword, axe, hammer, spear), bows, helmet/armor/boots, amulet/ring e special items. Os materiais usados são Wood, Stone, Copper/Common Metal, Bronze/Common Metals, Silver, Iron, Steel, Mythril e Adamantine; Bone é usado em acessórios e Leather em armaduras. Essa versão já condensa o núcleo lógico do jogo inteiro: mapa, biomas, leis, eras, criaturas, evolução, reprodução, comida, trabalho, livros, cultura, linguagem, política, economia, barcos, combate, magia, doenças e equipamentos. Para ficar realmente “wiki de referência”, o próximo passo natural é separar isso em páginas menores por tema, porque o material oficial é enorme e a lista completa de detalhes individuais espalha-se por centenas de páginas. 

Sim — pensando em refazer o jogo do zero até o nível atual, as duas listas que eu te dei ainda deixam faltar camadas inteiras da simulação. A wiki oficial não é só “biomas, criaturas e reinos”: ela também separa páginas de Status Effects, Unit Stats, Knowledge, Food, Happiness, World Laws, Gene Editor, Combat Actions and Spells, Diseases, Metamorphosis, Monolith and Evolution, Equipment, Boats, Jobs, Neurons, Phenotypes, Powers e Menu Art, além de Debug Menu, Hotkeys, Modding e Hidden and Unused Content. 

O que ficou mais faltando, na prática, foi isto:

1) Sistema de progressão e interface Faltaram páginas e mecânicas de Knowledge, Achievements, Hotkeys, Debug Menu, Menu Art, World History e toda a camada de leitura/controle do mundo. Isso é importante num remake porque define como o jogador entende o estado da simulação e como interage com ela. 

2) Genética, evolução e “biologia interna” As listas anteriores não cobriram direito Gene Editor, Phenotypes, Creature Genomes and Stats, Creature Default Meta-Traits, Creature Hierarchy, Creature Taxonomy and Collectives, Creature Sizes, Creature Aesthetics e Reproduction. A wiki oficial trata isso como uma das bases do comportamento das espécies, não só como detalhe de lore. 

3) Toda a camada de status e efeitos Faltaram Status Effects, Happiness, Diseases, além de como fome, ferimentos, silêncio, regeneração, cura, corrupção, loucura e estados especiais afetam unidades. Esses sistemas são centrais porque mudam combate, reprodução, construção e política. 

4) Estrutura social completa Eu citei reinos, clãs e alianças, mas ainda faltou amarrar a lógica de Villages, War, Religions, Cultures, Languages, Families, Subspecies, Loyalty, Influence, Kingdom Opinions, Kings and Leaders, Clan Traits, Kingdom Traits e Plots como um sistema único. A wiki separa tudo isso em páginas próprias porque cada uma altera a simulação de um jeito diferente. 

5) Conteúdo material e construção As listas ficaram muito amplas, mas não fecharam o catálogo de Buildings, Resources, Equipment, Boats, Jobs, Food, Economy, Common Metals, Gems, Gold, Leather, Bone, Books, Markets, Libraries, Mines, Barracks, Docks e as regras específicas de cada tier/uso. Para um remake fiel, essa parte precisa ser modelada como sistema, não só como inventário de itens. 

6) Combate e destruição em detalhe Também faltou a camada de Combat Actions and Spells, Destructive, Disasters, Explosives, projéteis, armas especiais e a lógica de dano/alcance/cooldown. A wiki oficial separa isso porque o combate em WorldBox não é só “unidade bate em unidade”; ele mistura magia, projéteis, efeitos de status e destruição de terreno. 

7) Conteúdo escondido, antigo e removido Um remake “até hoje” também precisa considerar o que a wiki marca como Removed Creatures, Unused Creatures, Hidden and Unused Content e páginas antigas de versão. Isso inclui coisas que o jogo atual não spawna no vanilla, mas que existem nos arquivos ou em builds antigas. 

8) Lista completa de criaturas ainda não estava fechada como catálogo Mesmo na parte das criaturas, a lista precisa ser organizada por categorias oficiais: a wiki diz que há 118 criaturas distintas na versão 0.51, separadas em Basic Creatures, Evolved Creatures, Other Creatures, Removed Creatures e Unused Creatures. Para um remake fiel, isso precisa virar uma tabela completa com comportamento, spawn, traits, evolução e função de cada uma. 

9) Powers e leis do mundo em totalidade Eu resumi poderes e leis, mas para um remake fiel faltam os detalhes de todas as abas e interações: a wiki diz que há cerca de 374 powers em 8 tabs, e o conjunto de World Laws altera regras duras do mundo. Isso é um sistema inteiro, não um menu secundário. 

Em uma frase: as listas anteriores cobriram o “macro” do WorldBox, mas ainda faltam o “micro” e o “engine layer” — genética, status, UI, debug, progressão, construção detalhada, combate detalhado, conteúdo oculto e o catálogo completo de unidades/itens/poderes. 

Se eu tivesse que transformar isso num checklist de remake, o próximo passo seria separar em três blocos: simulação base, conteúdo do jogo, e sistemas meta/ocultos.
